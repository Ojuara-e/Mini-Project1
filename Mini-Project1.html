1. Definio do Problema de Negcio

1.1. O Problema de Negcio

Nossa loja de e-commerce est em fase de crescimento, registrando um volume cada vez maior de transaes dirias. No entanto, essa grande quantidade de dados de vendas, em seu estado bruto, como um ba de tesouro trancado: sabemos que h valor ali, mas no conseguimos acess-lo.

Atualmente, muitas de nossas decises estratgicas so baseadas em intuio e observaes parciais, o que nos leva a enfrentar os seguintes desafios:

  • Gesto de Estoque Ineficiente: No temos clareza sobre quais produtos so nossos "campees de venda" e quais esto parados nas prateleiras. Isso resulta em excesso de estoque de itens de baixa procura e falta de produtos de alta demanda.

  • Marketing com Baixo Retorno: Nossas campanhas de marketing so genricas, pois no sabemos quais categorias de produtos atraem mais os clientes ou em quais regies geogrficas nosso pblico est mais concentrado.

  • Perda de Oportunidades Sazonais: No conseguimos identificar padres ou tendncias de vendas ao longo dos meses. Isso nos impede de planejar promoes estratgicas para perodos de alta ou de criar aes para impulsionar as vendas em meses de baixa.

  • Expanso sem Direo: Temos o desejo de expandir, mas no sabemos quais mercados regionais so mais promissores ou onde nossos esforos logsticos deveriam ser focados.

O problema central a falta de visibilidade clara sobre a performance do negcio, o que nos impede de tomar decises rpidas, inteligentes e baseadas em evidncias.

1.2. Objetivos do Projeto

Este projeto de anlise de dados visa transformar nossos dados brutos de vendas em insights acionveis. O objetivo responder a quatro perguntas de negcio fundamentais:

  • O que vender? Identificar os produtos de maior sucesso para otimizar nosso portflio e estoque.

  • Onde focar? Compreender quais categorias de produtos geram a maior parte da nossa receita.

  • Quando agir? Analisar a performance de vendas ao longo do tempo para identificar tendncias, picos e sazonalidades.

  • Para onde expandir? Mapear a distribuio geogrfica de nossas vendas para descobrir nossos mercados mais fortes.

1.3. Soluo Proposta

A soluo consiste em consolidar, limpar e analisar o histrico de dados de vendas da nossa plataforma. Utilizando ferramentas de anlise de dados (como Python com Pandas, NumPy e Matplotlib), vamos processar essas informaes e criar um relatrio visual que apresente as descobertas de forma clara e intuitiva para as equipes de gesto, marketing e operaes.

1.4. Resultados Esperados e Benefcios de Negcio

Ao final deste projeto, esperamos alcanar os seguintes resultados:

  • Otimizao de Estoque: Com a lista dos produtos mais e menos vendidos, poderemos ajustar nossas compras, reduzir custos com armazenamento e evitar a perda de vendas por falta de produto.

  • Marketing Direcionado e Eficaz: Sabendo quais categorias e regies so mais lucrativas, a equipe de marketing poder criar campanhas segmentadas, aumentando o retorno sobre o investimento (ROI).

  • Planejamento Estratgico: A visualizao das tendncias mensais permitir um melhor planejamento financeiro, promocional e de recursos, antecipando perodos de alta e baixa demanda.

  • Decises Baseadas em Dados: Substituiremos a intuio por dados concretos, criando uma cultura orientada a dados que impulsionar o crescimento sustentvel do negcio.

# Definio da funo para gerar dados fictcios de vendas
def gera_dados_ficticios(num_registros = 600):
    
    """
    Gera um DataFrame do Pandas com dados de vendas fictcios.
    """

    # Mensagem inicial indicando a quantidade de registros a serem gerados
    print(f"\nIniciando a gerao de {num_registros} registros de vendas...")

    # Dicionrio com produtos, suas categorias e preos
    produtos = {
        'Laptop Gamer': {'categoria': 'Eletrnicos', 'preco': 7500.00},
        'Mouse Vertical': {'categoria': 'Acessrios', 'preco': 250.00},
        'Teclado Mecnico': {'categoria': 'Acessrios', 'preco': 550.00},
        'Monitor Ultrawide': {'categoria': 'Eletrnicos', 'preco': 2800.00},
        'Cadeira Gamer': {'categoria': 'Mveis', 'preco': 1200.00},
        'Headset 7.1': {'categoria': 'Acessrios', 'preco': 800.00},
        'Placa de Vdeo': {'categoria': 'Hardware', 'preco': 4500.00},
        'SSD 1TB': {'categoria': 'Hardware', 'preco': 600.00}
    }

    # Cria uma lista apenas com os nomes dos produtos
    lista_produtos = list(produtos.keys())

    # Dicionrio com cidades e seus respectivos estados
    cidades_estados = {
        'So Paulo': 'SP', 'Rio de Janeiro': 'RJ', 'Belo Horizonte': 'MG',
        'Porto Alegre': 'RS', 'Salvador': 'BA', 'Curitiba': 'PR', 'Fortaleza': 'CE'
    }

    # Cria uma lista apenas com os nomes das cidades
    lista_cidades = list(cidades_estados.keys())

    # Lista que armazenar os registros de vendas
    dados_vendas = []

    # Define a data inicial dos pedidos
    data_inicial = datetime(2026, 1, 1)

    # Loop para gerar os registros de vendas
    for i in range(num_registros):
        
        # Seleciona aleatoriamente um produto
        produto_nome = random.choice(lista_produtos)

        # Seleciona aleatoriamente uma cidade
        cidade = random.choice(lista_cidades)

        # Gera uma quantidade de produtos vendida entre 1 e 7
        quantidade = np.random.randint(1, 8)

        # Calcula a data do pedido a partir da data inicial
        data_pedido = data_inicial + timedelta(days = int(i/5), hours = random.randint(0, 23))

        # Se o produto for Mouse ou Teclado, aplica desconto aleatrio de at 10%
        if produto_nome in ['Mouse Vertical', 'Teclado Mecnico']:
            preco_unitario = produtos[produto_nome]['preco'] * np.random.uniform(0.9, 1.0)
        else:
            preco_unitario = produtos[produto_nome]['preco']

        # Adiciona um registro de venda  lista
        dados_vendas.append({
            'ID_Pedido': 1000 + i,
            'Data_Pedido': data_pedido,
            'Nome_Produto': produto_nome,
            'Categoria': produtos[produto_nome]['categoria'],
            'Preco_Unitario': round(preco_unitario, 2),
            'Quantidade': quantidade,
            'ID_Cliente': np.random.randint(100, 150),
            'Cidade': cidade,
            'Estado': cidades_estados[cidade]
        })
    
    # Mensagem final indicando que a gerao terminou
    print("Gerao de dados concluda.\n")

    # Retorna os dados no formato de DataFrame
    return pd.DataFrame(dados_vendas)

10. Concluso e Entrega do Resultado

Existem vrias formas de entregar um projeto de anlise de dados e a escolha depende do pblico, do contexto e dos objetivos. Trs formas bastante utilizadas so:

10.1. Relatrio tcnico ou executivo (PDF, DOCX, etc.)

Essa forma clssica e muito til quando o pblico precisa de um documento formal para consulta. O relatrio pode conter descrio da metodologia, explorao dos dados, grficos, tabelas e concluses. comum separar a linguagem: uma verso mais tcnica (com cdigo, estatsticas detalhadas e testes) e outra mais executiva (com foco em insights, recomendaes e storytelling de dados).

10.2. Dashboard interativo (Power BI, Tableau, Looker, Streamlit, Dash, etc.)

Um dashboard permite que os usurios explorem os dados por conta prpria, filtrando informaes, ajustando perodos de tempo ou focando em variveis especficas. Essa forma de entrega muito valorizada em ambientes corporativos, pois facilita a tomada de deciso contnua e no exige conhecimentos tcnicos avanados dos usurios finais.

10.3. Apresentao (slides em PowerPoint, Google Slides, etc.)

Ideal para reunies de stakeholders, a entrega em formato de apresentao resume os principais pontos do projeto.
